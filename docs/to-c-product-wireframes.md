# DSA To C 化关键页面线框

本文档配合 [`docs/to-c-product-plan.md`](./to-c-product-plan.md) 输出 Phase 0 的 UI 线框，覆盖：

1. 登录 / 注册（`/login`、`/register`）— 已落地
2. 找回密码 / 重置密码（`/forgot-password`、`/reset-password`）— 已落地
3. 账户设置（`/account`）— 已落地（Phase 2）
4. 会员中心 / 升级（`/billing`）— 已落地（套餐对比、兑换码、支付弹窗、订单 / 发票入口）
5. 模型偏好（`/account`）— 已落地（用户从管理员配置且套餐允许的模型中选择）
6. 顶栏配额条（全局组件）— 已落地（位于 `SidebarNav` 内的 `QuotaIndicator`，复用桌面 / 移动端）
7. 超额引导对话框（全局组件）— 已落地（`QuotaExceededDialog` 监听 `dsa:quota-exceeded` 事件）
8. 首次引导（`/onboarding`，注册后添加自选股）— 已落地

> 线框使用 ASCII 表达布局，重点标注：组件来源、关键文案、调用的 API、关键交互。
> **不**包含像素级视觉规范；视觉细节沿用现有 `apps/dsa-web/src/components/common/` 中的 `Button` / `Input` / `Shell` / `SettingsAlert` 等基础组件，色彩沿用 `index.css` 中的 token。

## 1. 登录 / 注册（`/login`、`/register`）

> **状态**：已落地，对应 `apps/dsa-web/src/pages/UserAuthPage.tsx`。
> 本节作为后续迭代的对齐基线。

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│        ┌─────────────────────────────────────┐               │
│        │  [登录到 DSA]                        │               │
│        │  使用你的邮箱和密码登录, 享受 AI 自选股分析。 │       │
│        │                                     │               │
│        │  邮箱      [you@example.com      ]  │               │
│        │  密码      [••••••••••••       ]  │               │
│        │                                     │               │
│        │  [        登录 (主按钮)           ] │               │
│        │                                     │               │
│        │  忘记密码？        没有账号？立即注册 │               │
│        └─────────────────────────────────────┘               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**注册态新增字段**：

```
确认密码  [••••••••••••       ]
邀请码    [仅当 USER_INVITE_CODES 非空时显示]
□ 我已阅读并同意《服务条款》《隐私政策》《投资风险免责声明》
```

**关键交互**：

- **API**：`POST /api/v1/account/login` / `POST /api/v1/account/register`，详见 [`docs/to-c-mode.md`](./to-c-mode.md) §3。
- **登录成功**：根据 `redirect` 查询参数返回；默认 `/`。
- **注册成功**：若 `USER_REQUIRE_EMAIL_VERIFICATION=true`，停留在当前页提示「请前往邮箱完成验证」；否则直接登录并跳转。
- **错误展示**：使用 `SettingsAlert variant="error"`，文案优先取后端 `ParsedApiError.message`。
- **提交锁**：`isSubmitting` 期间按钮文案改为「登录中…」/「注册中…」并禁用。

**后续可选增强**：

- 第三方登录（微信扫码 / GitHub）只在确认运营策略后再加；MVP 不做。

## 2. 找回密码 / 重置密码

> **状态**：已落地，对应 `apps/dsa-web/src/pages/ForgotPasswordPage.tsx`。

**Step 1 — 发送重置邮件 (`/forgot-password`)**：

```
┌─────────────────────────────────────┐
│  [找回密码]                          │
│  输入注册邮箱, 我们会发送重置链接。     │
│                                     │
│  邮箱  [you@example.com    ]        │
│                                     │
│  [   发送重置邮件 (主按钮)        ]  │
│                                     │
│  返回登录                            │
└─────────────────────────────────────┘
```

- **API**：`POST /api/v1/account/request-password-reset`。
- **邮箱不存在**：后端不暴露存在性，前端统一显示「若该邮箱已注册, 重置邮件已发送, 请查收」。

**Step 2 — 用 token 重置（点击邮件后跳转 `/forgot-password?token=xxx`）**：

```
┌─────────────────────────────────────┐
│  [重置密码]                          │
│  请设置新密码。                       │
│                                     │
│  新密码      [••••••••••••     ]    │
│  确认密码    [••••••••••••     ]    │
│                                     │
│  [   重置并登录 (主按钮)         ]  │
└─────────────────────────────────────┘
```

- **API**：`POST /api/v1/account/reset-password`，成功后吊销该用户所有 session 并自动登录新 session。

│                                                             │

**关键交互**：

- **API**：
  - 读：`GET /api/v1/account/me`
  - 改密码：`POST /api/v1/account/change-password`（已上线，会吊销当前 session 并要求重新登录）
  - 通知偏好：`GET /api/v1/account/notification-prefs` / `PATCH /api/v1/account/notification-prefs`
- **通知偏好**：账户页已包含每日推送、邮件通知和 Pro Webhook 配置；免费档不能开启 AI 分析报告邮件推送。
- **自选股**：账户页提供快捷入口，独立 `/watchlist` 页面用于管理自选股列表。
- **退出登录**：账户页底部保留退出登录入口；自助注销账号与个人数据导出入口暂不在普通账户页展示。

```
┌─────────────────────────────────────────────────────────────┐
│ [会员中心]                                                   │
│ 当前套餐: 免费会员 · 不会过期                                  │
│ ▌升级到 Pro 解锁: 30 只自选股 · 50 次/天分析 · webhook 推送    │

│  [免费]            [Pro 月付]            [Pro 年付]          │
│  ¥0 / 永久          ¥XX / 月              ¥XXX / 年          │
│                                                             │
│  · 3 只自选股        · 30 只自选股          · 30 只自选股     │
│  · 5 次/天 分析      · 50 次/天 分析        · 50 次/天 分析   │
│  · 5 次/天 Agent     · 50 次/天 Agent       · 50 次/天 Agent  │
│  · 无报告推送        · 邮箱 + Webhook       · 邮箱 + Webhook  │
│  · 平台基础模型      · 平台高级模型          · 平台高级模型    │
│  · 可选基础模型      · 可选高级模型          · 可选高级模型    │
│                                                             │
│  [当前档位]          [→ 升级 (主按钮)]      [→ 升级 (推荐)]  │
├─────────────────────────────────────────────────────────────┤
│ ─── 兑换码 ───                                               │
│ [        请输入兑换码        ]   [   兑换    ]                │
│                                                             │
│ ─── 订阅记录 ───                                              │
│ 2026-05-01  Pro 月付  来源: 兑换码 ABCD1234   到期: 2026-06-01│
│ 2026-04-01  Pro 月付  来源: 兑换码 WXYZ5678   到期: 2026-05-01│
└─────────────────────────────────────────────────────────────┘
```

**关键交互**：

- **API**：
  - `GET /api/v1/billing/plans`：列出当前可见套餐，价格 / 配额 / 能力开关。
  - `GET /api/v1/billing/subscription`：当前订阅 + 历史。
  - `POST /api/v1/account/redeem`：用兑换码换 Pro。
  - `POST /api/v1/billing/orders`：创建订单。
  - `POST /api/v1/billing/orders/{order_no}/pay`：拉起微信 / 支付宝支付，返回二维码内容。
  - `GET /api/v1/billing/orders/{order_no}`：支付弹窗每 2s 轮询订单状态。
  - `POST /api/v1/billing/orders/{order_no}/mock-pay`：仅 mock 支付模式下模拟支付成功。
- **MVP 路径**：免费 → 兑换码升 Pro，或通过 `PaymentDialog` 选择微信 / 支付宝扫码升级；`PAYMENT_MOCK_ENABLED=true` 时可不接真实通道完成联调。
- **降级**：Pro 到期未续费自动回 free，所有过限的自选股保留只读但不参与调度。
- **文案规则**：所有「分析次数」措辞统一为「次/天」，避免次/月混用。

## 5. 模型偏好（`/account` — 已落地）

```
┌─────────────────────────────────────────────────────────────┐
│ [模型偏好]                                                    │
│ 从平台管理员配置且当前套餐允许的模型中选择首选模型。              │
├─────────────────────────────────────────────────────────────┤
│  当前套餐     Pro 月付                                        │
│  首选模型     [ deepseek/deepseek-v4-flash ▼ ]                │

│                                                             │
│  [保存偏好] [恢复默认]                                        │
└─────────────────────────────────────────────────────────────┘
```

**关键交互**：

- **API**：
  - `GET /api/v1/account/model-preference`
  - `PATCH /api/v1/account/model-preference`
- **路由规则**：`src/users/model_router.py` 已接入实际 LLM 调用链；用户首选模型必须属于管理员配置且当前套餐允许的模型列表，否则回退平台默认模型顺序。
- **权限规则**：普通用户不能访问部署级模型渠道、API Key 或系统设置。
- **错误处理**：加载 / 保存失败时使用 `SettingsAlert variant="error"` 展示后端错误。

## 6. 顶栏配额条（全局组件，已落地）

挂在 `SidebarNav` 内，桌面侧边栏与移动抽屉复用，所有登录后的页面可见。

```
[今日剩余: 分析 2/5 · Agent 4/5]
```

**变体**：

```
免费档剩余正常:    [今日剩余: 分析 2/5 · Agent 4/5]               (中性色)
免费档已用完:      [今日剩余: 分析 0/5 · Agent 0/5  → 升级]       (警告色, 点击 → /billing)
Pro 档剩余正常:    [今日剩余: 分析 35/50 · Agent 42/50]            (中性色)
Pro 档已用完:      [今日剩余: 分析 0/50 · Agent 0/50 → 升级]       (警告色, 点击 → /billing)
```

**关键交互**：

- **API**：`GET /api/v1/account/status` 返回当日 `analysis` / `agent` 用量和 plan 摘要。
- **刷新策略**：`AuthContext` 初始化拉取状态；分析、Agent、支付 / 兑换完成后按业务回调刷新。
- **响应式**：侧边栏 / 移动抽屉复用同一组件，避免桌面与移动端状态不一致。

## 7. 超额引导对话框（全局组件，已落地）

当业务接口返回 `quota_exceeded` 错误码时（Phase 2 配额服务新增），全局拦截并弹出。

```
┌────────────────────────────────────────────┐
│  ⚠ 今日额度已用完                            │
│                                            │
│  你今天的「AI 分析」额度已用完 (5/5)。        │
│  明天 0 点重置, 也可以选择以下方式立即继续：   │
│                                            │
│  [→ 升级到 Pro (推荐)]                      │
│  [关闭]                                     │
└────────────────────────────────────────────┘
```

**关键交互**：

- **触发**：业务 API 返回 HTTP 402 + `error.code='quota_exceeded'` + `error.kind='analysis'|'agent'|'notify'`。
- **拦截层**：axios interceptor 派发 `dsa:quota-exceeded` 事件，由根组件挂载的 `QuotaExceededDialog` 接管展示，避免每个页面重复实现。
- **升级按钮**：跳转 `/billing` 并带 `?from=quota&kind=analysis`。
- **模型提示**：若当前首选模型下架或不再属于套餐允许范围，账户页模型偏好卡片提示用户重新选择；实际请求自动回退到可用平台模型。

## 8. 首次引导（注册后自选股设置，已落地）

注册成功后进入 `/onboarding`，使用独立全屏引导页添加 1–3 只自选股：

```
┌─────────────────────────────────────────────────────────────┐
│  欢迎使用 DSA                                               │
│                                                             │
│  先选 1–3 只你最关心的股票，后续可用于每日 AI 分析报告。          │
│                                                             │
│  自选股  [+ 600519] [+ AAPL] [+ HK00700]                    │
│  搜索    [输入股票代码或名称…]                                 │
│                                                             │
│  [稍后再设置]                  [完成设置，开始体验 →]          │
└─────────────────────────────────────────────────────────────┘
```

**关键交互**：

- **入口**：注册成功后跳转 `/onboarding`；该路由仅在用户模式下可访问。
- **数据写入**：搜索添加股票时直接调用 `POST /api/v1/account/watchlist` 写入 per-user 自选股表。
- **数量限制**：显示 `min(plan.maxStocks, 3)` 个进度点；免费档默认最多 3 只。
- **跳过策略**：当前实现仅前端跳回首页，不写入独立 onboarded 标记。

## 9. 文案与合规要求

- **每页页脚**：固定显示「内容由 AI 生成, 仅供参考, 不构成投资建议。投资有风险, 入市需谨慎。」
- **每封邮件**：模板末尾追加同一句免责声明，并附 HMAC 一键退订链接（`GET /api/v1/account/notification-prefs/unsubscribe`）。
- **错误文案统一规范**：参考现有 `apps/dsa-web/src/api/error.ts`：
  - 网络错误：「网络异常, 请稍后重试」
  - 配额错误：「今日额度已用完」
  - 鉴权错误：「请先登录」
  - 服务端错误：「服务暂时不可用, 我们正在处理」
