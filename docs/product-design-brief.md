# DSA 产品设计需求描述

本文档面向产品设计师、UI/UX 设计师和外部视觉合作方，用于快速理解 DSA 当前产品、目标用户、核心任务、页面范围与设计交付要求。本文不是研发实现方案；接口、表结构和工程阶段以 [`to-c-product-plan.md`](./to-c-product-plan.md)、[`to-c-user-stories.md`](./to-c-user-stories.md)、[`to-c-product-wireframes.md`](./to-c-product-wireframes.md) 和 [`web-frontend-redesign-plan.md`](./web-frontend-redesign-plan.md) 为准。

## 1. 项目一句话定位

DSA 是一个面向个人投资者的「股票 AI 分析助手」：用户添加自选股后，系统聚合 A 股、港股、美股的行情、技术指标、新闻、公告、基本面与舆情信息，通过大模型生成结构化分析报告、风险提示、操作检查清单和每日推送，帮助用户更高效地做投资研究。

产品必须始终保持「辅助分析」定位，不输出保证收益、确定性荐股或诱导交易承诺。所有投资相关表达都应附带风险提示，避免让用户误解为持牌投顾建议。

## 2. 设计目标

- **专业可信**：让用户感到这是严肃的金融分析工具，而不是玩具型 AI Demo。
- **降低使用门槛**：新用户能在 1–3 分钟内完成注册、添加自选股并看到第一份报告。
- **突出核心任务**：首页优先服务「输入股票并生成分析报告」，问股页优先服务「围绕股票追问」。
- **数据可读**：报告、指标、历史记录、订单和表格必须清晰、可扫描、适合长时间阅读。
- **商业化清晰**：免费额度、Pro 权益、升级原因、支付状态、续费和退款路径要明确。
- **响应式可用**：桌面端是主场景，移动端至少保证完整浏览、分析提交、问股和账户管理可用。

## 3. 目标用户

### 3.1 个人投资者

- **特征**：关注 A 股、港股或美股；有自选股；希望每天快速掌握变化。
- **核心诉求**：想知道某只股票「现在发生了什么、风险在哪里、接下来重点看什么」。
- **设计关注**：降低术语负担，用层级和解释帮助用户理解指标，而不是堆砌专业数据。

### 3.2 进阶研究型用户

- **特征**：会看技术指标、趋势、新闻、资金流和历史报告；愿意使用 Agent 追问。
- **核心诉求**：希望把 AI 当作研究助理，快速整理信息、验证观点和沉淀报告。
- **设计关注**：提供完整 Markdown 报告、历史对比、策略选择、技能选择、回测和持仓视角。

### 3.3 付费 Pro 用户

- **特征**：希望跟踪更多自选股，需要每日自动推送、更高配额和高级模型。
- **核心诉求**：减少每天主动打开系统的成本，定时收到个人化报告。
- **设计关注**：权益展示、到期提醒、续费入口、订单/发票/退款路径要可靠、透明。

### 3.4 平台管理员

- **特征**：维护模型渠道、订单、退款、发票、公告和用户状态。
- **核心诉求**：能高效处理运营动作和异常排查。
- **设计关注**：后台不追求营销感，重点是表格效率、状态清晰、风险操作可确认。

## 4. 核心用户路径

### 4.1 新用户首次体验

1. 进入登录/注册页。
2. 使用邮箱注册并勾选服务条款、隐私政策和风险免责声明。
3. 完成邮箱验证或注册成功后进入首次引导。
4. 添加 1–3 只自选股。
5. 点击分析，看到首份 AI 决策报告。
6. 在报告页看到风险提示、评分、趋势、催化因素、操作检查清单和免责声明。

### 4.2 日常手动分析

1. 在首页搜索股票代码、名称或拼音。
2. 选择分析策略。
3. 提交分析任务。
4. 在任务区看到排队、运行、完成或失败状态。
5. 在报告工作区阅读报告。
6. 可查看历史报告、筛选历史、重新打开历史结果。

### 4.3 AI 问股

1. 进入问股页。
2. 选择或创建会话。
3. 选择分析技能或策略。
4. 输入问题，例如「这只股票短线风险是什么」。
5. 查看流式回答、引用的信息和后续追问建议。
6. 可复制、导出或发送会话内容。

### 4.4 订阅升级

1. 用户在配额不足、Pro 功能置灰或会员中心看到升级入口。
2. 进入会员中心，对比免费、Pro 月付、Pro 年付权益。
3. 选择套餐并使用微信或支付宝扫码支付。
4. 支付弹窗展示二维码、订单金额、轮询状态和失败重试。
5. 支付成功后立即刷新权益和配额。
6. 可在订单页申请退款，在发票页申请电子发票。

### 4.5 每日自动推送

1. Pro 用户在账户页维护自选股。
2. 开启每日推送、邮件通知或 Webhook。
3. 每个交易日傍晚收到个人化报告。
4. 邮件中可查看摘要、风险提示、免责声明和一键退订入口。

## 5. 页面范围与设计重点

| 页面 / 模块 | 目标 | 设计重点 |
| --- | --- | --- |
| 登录 / 注册 | 完成身份进入 | 信任感、低干扰、协议确认、错误提示清楚 |
| 首次引导 | 帮用户建立第一批自选股 | 步骤感、进度反馈、示例股票、完成后的明确去向 |
| 首页分析工作台 | 提交分析、查看任务和报告 | 股票搜索优先、报告阅读优先、历史和任务作为辅助区 |
| 报告详情 | 阅读 AI 分析结果 | 结论先行、风险醒目、Markdown 可读、长报告不压迫 |
| AI 问股 | 多轮投研对话 | 会话列表、消息流、技能选择、输入区层级清楚 |
| 持仓管理 | 管理账户、持仓、现金和风险 | 数据表格清晰、关键指标突出、录入流程不拥挤 |
| 回测 / 历史验证 | 验证策略表现 | 参数区与结果区分离，指标、图表、明细有明确层级 |
| 账户页 | 管理个人信息、自选股、通知和模型偏好 | 分区明确，普通用户不能看到部署级系统配置 |
| 会员中心 | 展示套餐并完成升级 | 权益对比强、推荐档突出、价格和限制透明 |
| 订单 / 发票 | 查看商业化记录 | 状态标签一致，金额、时间、操作入口清楚 |
| 公告 / 帮助 | 承载通知和客服入口 | 可读性、检索感、反馈路径明确 |
| 法律协议 | 公开阅读服务条款、隐私政策、风险披露 | 正文排版舒适，重点风险提示清楚 |
| 管理后台 | 运营处理用户、订单、退款、发票和公告 | 表格效率、批量视角、危险操作确认 |

## 6. 信息架构建议

主导航建议按用户任务分组，而不是按技术模块分组：

- **分析**：首页 / 报告 / 历史。
- **问股**：AI 对话与策略追问。
- **资产**：持仓、回测、告警。
- **账户**：账户设置、会员中心、订单、发票。
- **支持**：公告、帮助、法律协议。
- **管理**：仅管理员可见的运营后台。

普通 C 端用户不应看到系统部署配置、平台 API Key、全局模型渠道等管理员能力。若普通用户直接访问系统设置，应引导回账户页或显示无权限状态。

## 7. 视觉方向

推荐方向：现代金融智能工作台。

### 7.1 关键词

- 专业。
- 清爽。
- 低噪声。
- 数据优先。
- AI 辅助感明确。
- 适合长期阅读。
- 桌面宽屏友好，移动端不丢功能。

### 7.2 视觉原则

- **浅色优先，暗色兼容**：默认视觉不应过暗，暗色模式作为长期使用场景补充。
- **减少强装饰**：避免大面积霓虹、发光边框、玻璃拟态和赛博终端风。
- **金融状态色克制使用**：红、绿、黄只服务涨跌、风险、状态，不做大面积装饰背景。
- **主色用于行动**：蓝青色系可作为品牌和主操作色，但应降低饱和度。
- **留白建立秩序**：用卡片、边框、间距、标题和表格对齐建立信息层级。
- **动效服务反馈**：仅用于加载、弹层、导航切换、流式输出和状态变化。

## 8. 组件与交互要求

### 8.1 基础组件

设计稿中建议沉淀以下组件：

- Button：主按钮、次按钮、危险按钮、文本按钮、图标按钮、加载态、禁用态。
- Input / Select / Textarea：普通态、聚焦态、错误态、禁用态、说明文案。
- Card / Section：标准内容卡片、指标卡、报告卡、订单卡。
- Badge / Tag：套餐、订单状态、任务状态、风险等级、市场类型。
- Dialog / Drawer / Popover：支付、确认、历史筛选、移动端导航。
- Table：数值列、状态列、操作列、空态、加载态、分页。
- Alert / Notice：错误、警告、成功、续费提醒、配额不足提醒。
- Markdown / Prose：报告正文、法律协议、AI 回答。

### 8.2 状态覆盖

每个关键页面都需要设计以下状态：

- 初始空态。
- 加载中。
- 加载失败。
- 无权限。
- 未登录。
- 配额不足。
- Pro 权益不足。
- 表单校验失败。
- 提交中。
- 成功反馈。
- 长内容溢出和移动端折叠。

## 9. 响应式要求

### 9.1 桌面端

- 主要设计宽度建议覆盖 1440px 和 1280px。
- 左侧导航固定，主内容区独立滚动。
- 首页、问股、持仓和回测可采用工作台式宽屏布局。
- 标准内容页最大宽度建议控制在 1120px–1280px。

### 9.2 移动端

- 主要设计宽度建议覆盖 390px。
- 导航改为抽屉或底部可达入口。
- 首页历史记录、任务列表、筛选面板可收纳为 drawer / bottom sheet。
- 表格允许横向滚动，但关键摘要应先以卡片呈现。
- 支付二维码、协议阅读、问股输入区需要保证单手操作可用。

## 10. 文案与合规边界

### 10.1 推荐文案口径

- 使用「AI 分析」「风险提示」「辅助决策」「研究参考」「操作检查清单」。
- 使用「看多 / 震荡 / 看空」时，应作为模型判断或分析倾向展示，不包装为确定性买卖指令。
- 强调「不构成投资建议」「股市有风险，投资需谨慎」。

### 10.2 避免文案

- 避免「稳赚」「必涨」「抄底信号」「闭眼买入」。
- 避免「专家荐股」「老师带单」「收益保证」。
- 避免把评分直接解释为确定性收益概率。

## 11. 设计交付物要求

建议设计师交付以下内容：

1. **信息架构图**：导航分组、页面层级、普通用户和管理员入口差异。
2. **核心流程图**：注册引导、手动分析、问股、升级支付、订单退款、每日推送。
3. **高保真页面**：至少覆盖登录/注册、首次引导、首页、报告详情、问股、账户、会员中心、订单、帮助、管理员后台。
4. **响应式稿**：首页、报告详情、问股、会员中心至少提供桌面端和移动端版本。
5. **设计系统基础规范**：颜色、字体、间距、圆角、阴影、按钮、输入框、卡片、表格、状态标签、弹窗、Markdown 正文。
6. **状态稿**：空态、错误态、加载态、配额不足、支付中、支付成功、无权限、Pro 权益不足。
7. **切图 / 图标说明**：若使用自定义插画或图标，需要标明授权来源和可商用性。

## 12. 优先级建议

### P0：必须先设计

- 登录 / 注册 / 邮箱验证。
- 首次引导。
- 首页分析工作台。
- 报告详情 / Markdown 阅读体验。
- AI 问股。
- 账户页与配额展示。
- 会员中心与支付弹窗。

### P1：紧随其后

- 订单与退款。
- 发票申请。
- 帮助中心。
- 公告中心。
- 持仓管理。
- 回测 / 历史验证。

### P2：后台与增强

- 管理员后台。
- 运营统计和用量看板。
- 移动端细节优化。
- 邮件模板视觉。

## 14. 设计评审检查清单

- 是否一眼能看出产品是股票 AI 分析助手。
- 是否避免了确定性荐股和收益承诺。
- 新用户是否能顺利完成注册、添加自选股和首次分析。
- 首页是否把股票搜索和分析结果放在最高优先级。
- 报告是否适合长时间阅读，风险提示是否足够醒目。
- 免费与 Pro 的权益差异是否清楚。
- 配额不足、无权限、支付失败等状态是否有明确下一步。
- 普通用户是否看不到部署级系统设置。
- 移动端是否能完成核心路径。
- 组件样式是否统一，是否能沉淀为长期设计系统。
